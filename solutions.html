ArcGIS Experience Builder Developer Edition

To explore the power of ArcGIS Experience Builder, ArcGIS Experience Builder Developer Edition was installed on James and Kristine's computers. Since the server can be installed using a personal ArcGIS Developers account, this offers a testing environment for custom development that is not connected to a larger organization. Furthermore, experiences built with the Developer Edition will not disappear once a user is no longer part of an organization. Drawbacks include having to start the client and server every time you want to use the Developer Edition, inability to share an experience easily with external users and having to learn TypeScript and React to create custom widgets. Therefore, if the client does not have a requirement for custom development, there is no need to install ArcGIS Experience Builder Developer Edition and ArcGIS Experience Builder will be able to meet all the project requirements.

Installing the Experience Builder Developers Editions is no mean feat! The listed instructions breeze through certain specific steps, such as the need to go to the Developers account OAuth2.0 tab and reset the redirect-URI to localhost:3001 in order to be able to connect with the actual development tool once everything has been installed. It requires download and install of Node.js which needs a system reboot. And then after the download and install of the primary package (ArcGISExperienceBuilder) it requires activation of the client and server components in Command Prompt, and then these need to be turned on with the command prompt every time it is used. As we were developing Experience options for our collaborative project's dynamic map, it seemed that a custom widget may be require to search through the data and select stream segments by NextDownIDs, and so we did explore the custom widget direction, but now are moving more in the direction of restructuring the data in the final WebMap to fascilitate this functionality.

ArcGIS Hub

While ArcGIS Experience Builder has the ability to create web applications that look like webpages (i.e. an experience can contain a page with all the front-end web design elements), the client did not specify that everything must be hosted in Experience Builder. A Hub Site Application was developed with ArcGIS Hub to create a web application that resembles a webpage. Advantages of Hub include more customization options available for the layout and seamless integration of ArcGIS applications like web maps, web experiences and graphs.
